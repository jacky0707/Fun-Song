前言

React-three-fiber 官網:

react-three-fiber is a React renderer for threejs.

React 官網:

Renderers manage how a React tree turns into the underlying platform calls.

最初是因為當時 VR 跟元宇宙的話題正熱的飛起，所以想說來找找有沒有類似 3D 世界的東西來玩玩看。而 react-three-fiber 以及裡面的 live-demo 就這樣映入我的眼簾。

簡單的 hooks 的使用以及不難理解的 jsx 語法就能打造出這樣的 3D 場景，這個東西很快地就吸引了我。

React-three-fiber 官網建議對 React 以及 Three.js 有一定基礎的認識。 React 的部分 hooks 尤其重要，Three.js 的部分建議至少有些 basic grasp，並且看過上面提到的 live-demo。

我個人 React 是熟悉的，但 Three.js 的部分則是一股腦地衝進去後撞得頭破血流再開始一點一點的補坑……

react-three-fiber 生態系

3D 網頁的專案跟以往做的網頁比起來多了不少新的概念， react-three-fiber 也列出了一個 Ecosystem 來讓我們可以輕鬆地找到適當的工具。在稍微瀏覽過這些的文件後，決定簡單使用下列:

  1. @react-three/drei – useful helpers for react-three-fiber
  2. @react-three/cannon – useful helpers for react-three-fiber
  3. zustand – state management

zustand 的部分比較有趣，以往我都是使用 redux 來做狀態管理，但這次剛好有機會認識這個正在崛起的狀態管理庫，學習新東西擇日不如狀日，直接入坑一波~

目標 & 預期實現方式

  • 讓使用者宛如置身一個 3D 世界
    • 建立一個 3D scene，包含天空、地板、光線、光源。 — react-three/fiber 建立場景。
    • 應有基本物理，碰撞、重力等等。 — react-three/cannon。
    • 3D 世界中簡單的平面、方形、圓形物體。 — react-three/cannon usePlane, useBox, useSphere。
  • 可以在裡面仰頭、俯首。
    • 預期用滑鼠控制視角。 — react-three/drei 的 pointerlock control。
  • 可以前後左右自由行走。
    • 預期用鍵盤按鍵控制自身移動方向。 — react-three/connon 建立本體實例，再使用 event listener 去控制實例。

建立一個 3D 世界

使用 react-three/fiber 的 Canvas 來建立 3D 場景。參考 Canvas。

  • Canvas 為定義整個 3D 場景的物件
  • Sky 天空物件
  • ambientLight 為環境光源
  • pointLight 為點光源，可以想像成太陽
  • Physics 是 react-three/cannon 提供的物理系統
<Canvas
  id="canvas"
  raycaster={{
    // 設定 offset，畫面大小變化時滑鼠可以讓滑鼠位置水平垂直置中
    computeOffsets: (_, { size: { width, height } }) => {
      return {
        offsetX: width / 2,
        offsetY: height / 2,
      };
    },
  }}
>
  <Sky distance={450000} sunPosition={new Vector3(100, 10, 100)} />
  <ambientLight intensity={1} />
  <pointLight castShadow intensity={0.5} position={(100, 100, 100)} />
  <Physics gravity={[0, -30, 0]}>
    <Ground />
  </Physics>
</Canvas>

Ground Component 程式碼。參考 Loading texture。

  • usePlane 是 react-three/cannon 提供的其中一種 hooks ，可以在 Canvas 內新增一個平面，而解構出來的 ref 以及 api 則提供屬性以及函數來讓我們使用。
  • useLoader 是 react-three/drei 提供的 helper，幫助我們更簡易的套用我們希望使用的 texture。
  • mesh 是 react-three/fiber 中常見綁定 ref 以及給予樣式的方式，參考 mesh 給予適當的 geometry 跟 material 就能夠調整 mesh 的樣式。
const Ground = (props) => {
  const [ref, api] = usePlane(() => ({
    rotation: [-Math.PI / 2, 0, 0],
    ...props,
  }));

  const texture = useLoader(THREE.TextureLoader, "/ground.jpg");
  texture.wrapS = texture.wrapT = THREE.RepeatWrapping;
  texture.repeat.set(4500, 4500);

  return (
    <mesh ref={ref} receiveShadow>
      <planeBufferGeometry attach="geometry" args={[1000, 1000]} />
      <meshStandardMaterial map={texture} />
    </mesh>
  );
};

使用滑鼠控制視角

參考drei 原始碼，滑鼠控制視角例子，基於這兩個例子，我希望我的 pointerLock control 可以多一個用來解除 lock status 的 state，這有利於我 side project 的開發，因此我將原始碼稍做了修改

const PointerLockControls = React.forwardRef(() =>
  // ...
  {
    const { camera, lockState, ...rest } = props;

    // ...

    React.useEffect(() => {
      if (!lockState) {
        controls.unlock();
      }
    }, [lockState]);

    // ...

    return <primitive ref={ref} object={controls} {...rest} />;
  }
);

將修改過的 PointerLockControls 套用給 User。

const User = (props) => {
  const { lockState, setLockState } = props;

  // user Sphere
  const { camera } = useThree();
  const [ref, api] = useSphere(() => ({
    mass: 1,
    type: "Dynamic",
    position: [0, 1, 0],
  }));

  // ...

  return (
    <>
      <mesh ref={ref}>
        <PointerLockControls
          lockState={lockState}
          onLock={() => setLockState(true)}
          onUnlock={() => setLockState(false)}
          selector="#canvas"
        />
      </mesh>
    </>
  );
};

前後左右自由行走

這部分參考了 Building JavaScript Minecraft in 1 Hour [React & Three.js Tutorial] 以及 I made MINECRAFT using REACT ，這兩部影片用的都是稍舊版本的 react-three/fiber，因此可能會遇到一些小坑，但是基本的概念都是一樣的。

首先要註冊鍵盤的 wasd 按鍵事件，這邊就參考上列影片將他們包成 hooks

const moveFieldByKey = (key) => {
  const keys = {
    KeyW: "moveForward",
    KeyS: "moveBackward",
    KeyA: "moveLeft",
    KeyD: "moveRight",
    Space: "jump",
    ShiftLeft: "speedUp",
  };
  return keys[key];
};

export const useKeyboardControls = () => {
  const [movement, setMovement] = useState({
    moveForward: false,
    moveBackward: false,
    moveLeft: false,
    moveRight: false,
    jump: false,
    speedUp: false,
  });

  useEffect(() => {
    const handleKeyDown = (e) => {
      setMovement((m) => ({
        ...m,
        [moveFieldByKey(e.code)]: true,
      }));
    };

    const handleKeyUp = (e) => {
      setMovement((m) => ({
        ...m,
        [moveFieldByKey(e.code)]: false,
      }));
    };

    document.addEventListener("keydown", handleKeyDown);
    document.addEventListener("keyup", handleKeyUp);

    return () => {
      document.removeEventListener("keydown", handleKeyDown);
      document.removeEventListener("keyup", handleKeyUp);
    };
  }, []);

  return movement;
};

接著在 User 中使用，並且在按下按鍵時套用速度。

const User = (props) => {
  // ...

  // get movement
  const { moveForward, moveBackward, moveLeft, moveRight, jump, speedUp } =
    useKeyboardControls();

  // initial velocity
  const velocity = useRef([0, 0, 0]);

  useFrame(() => {
    if (lockState) {
      // get direction
      const direction = new Vector3();
      // horizontal direction, z-direction
      const frontVector = new Vector3(
        0,
        0,
        Number(moveBackward) - Number(moveForward)
      );
      // horizontal direction, x-direction
      const sideVector = new Vector3(
        Number(moveLeft) - Number(moveRight),
        0,
        0
      );

      // vertical, y-direction
      const SPEED = speedUp ? 9 : 3;

      // apply horizontal movement to direction
      direction
        .subVectors(frontVector, sideVector)
        .normalize()
        .multiplyScalar(SPEED)
        .applyEuler(camera.rotation);

      api.velocity.set(direction.x, velocity.current[1], direction.z);

      if (jump && Math.abs(velocity.current[1].toFixed(2)) < 0.01) {
        api.velocity.set(velocity.current[0], 6, velocity.current[2]);
      }
    } else {
      api.velocity.set(0, velocity.current[1], 0);
    }
  });

  //...
};

最後就是將 position 這個 ref 去 subscribe useSphere 所 return 的 api，並且 camera 的位置也使用 camera.position.copy(position.current) 來跟 sphere 保持同樣位置。

//...

// position ref
const position = useRef(new Vector3(0, 0, 0));

useEffect(() => {
  const unsubscribePosition = api.position.subscribe(
    (pos) => (position.current = new Vector3(pos[0], pos[1], pos[2]))
  );
  return () => {
    unsubscribePosition();
  };
}, []);

useFrame(() => {
  camera.position.copy(position.current);

  if (lockState) {
    //...
  }
});

這邊有個曾經讓我卡住很久的點是， camera(我們看到的視角) 跟 sphere(我們使用 useSphere 做出來的球體) 本身其實是不同的東西，ref 跟 api 都是屬於 sphere 的東西。

如果最後沒有使用 subscribe 跟 camera.position.copy，那麼即使 sphere 本身有套用到第二步所做的，按下按鍵改變移動速度，相機本身沒有跟隨球體移動，那我們看到的視角跟畫面都會是一樣的。

這部分如果沒辦法理解的話，可以先註解掉嘗試在 User 的 mesh 中增加 sphereGeometry，然後取消 subscribe 或 camera.position.copy，這樣就可以體會 camera 跟球體分離的情況了。

<mesh ref={ref}>
  <sphereGeometry />
  <PointerLockControls />
</mesh>

結語

第一次開發 3D 專案，真的是撞得頭破血流，但也能體會為什麼元宇宙這種東西這麼吸引人。簡單的小專案，就可以感覺到無限的可能性，可惜身邊沒有美術專家，純工程師做的 3D 場景只可能有方形、圓形、平面這種簡單到不行的模型，期待之後可以將這個 side project 發揚光大。
